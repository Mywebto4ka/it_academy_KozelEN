Формат WebP разработан в 2010 году компанией Google.

Поддерживает сжатие с потерями и без, а также полупрозрачность и анимацию. 
Он считается одним из лучших, поскольку дает значительное улучшение производительности сайта.

По результатам исследования Google:

  • Картинка в WebP, сжатая без потерь, меньше PNG на 26%.
  • - Картинка в WebP, сжатая с потерями, меньше JPEG на 25-34%.
  • - WebP поддерживает прозрачность без потерь (известную, как альфа-канал) при увеличении размера всего лишь на 22%.


Недостаток: 
Новые изображения займут дополнительное место на хостинге, хотя они будут в несколько раз меньше оригиналов. Решение. Увеличение места на сервере.
Не все браузеры поддерживают этот формат. В настоящее время нет никаких проблем с WebP в Google Chrome, Opera, Firefox и др. Решение. Возможна реализация, когда при невозможности отображения браузером подгружается .jpg.
Файл WebP на ПК можно открыть только некоторыми программами. Т.е. картинки будет сложнее использовать, следовательно больший процент уникального контента на сайте)) 
При сжатии с потерями изображение может выглядеть плоским, как будто «пластиковым».Решение. Не использовать сжатие меньше 80%.

Какие браузеры поддерживают WebP

В настоящее время формат поддерживается следующими браузерами (информация актуальна на момент написания статьи, однако может измениться со временем).

  • Google Chrome: WebP поддерживается браузером с версии 2.
  • Mozilla Firefox: он начал поддерживать WebP в версии 65.
  • Microsoft Edge: он начал поддерживать WebP с версии 18, но теперь Edge основан на Chromium и поддерживает WebP.
  • Apple Safari: не имеет встроенной поддержки WebP. Однако его можно просматривать в Safari, используя плагин или конвертируя изображения в другой формат.
  • Opera: поддерживает WebP с версии 11.
  • Android Browser: поддерживает WebP с версии 4.0 (Ice Cream Sandwich).
  • Chrome для Android: работает с WebP начиная с версии 25.
  • Samsung Internet Browser: поддерживает WebP с версии 4.0.
